# 🤖 Clean-Clean Blog Agent — Інструкція

Це файл для автоматичного агента, що пише статті в блог щотижня.

## Як працює

1. Агент запускається раз на тиждень (понеділок, 10:00 UTC).
2. Тягне дані з Google Search Console (через файл `gsc_report.json` в репо).
3. Перевіряє `BLOG_TOPICS.md` — список тем у черзі.
4. Якщо в GSC є запит з показами 10+ і позицією 11–30 — пише статтю **під цей запит**.
5. Якщо ні — бере **наступну тему з черги** в `BLOG_TOPICS.md`.
6. Пише повну статтю (1500–2500 слів) у `blog/<slug>.html`.
7. Оновлює `blog.html` (додає картку нової статті у топ).
8. Оновлює `sitemap.xml`.
9. Робить commit + push на GitHub.
10. Позначає тему як ✅ done у `BLOG_TOPICS.md`.

## Стиль статей

- **Мова:** українська
- **Тон:** експертний, без води, конкретні цифри
- **Довжина:** 1500–2500 слів
- **Структура:** H1 → інтро → H2 (5–8 секцій) → CTA → Related articles
- **Внутрішні посилання:** 3–5 на сторінки послуг або міст
- **Schema.org:** Article з повними полями
- **OG:** image, title, description
- **Брендинг:** Clean-Clean, темний navy + золоті акценти, btn-3d сині кнопки

## Ключові правила

1. ❌ НЕ вигадуй ціни які не відповідають прайсу Clean-Clean
2. ❌ НЕ обіцяй послуг яких немає на сайті
3. ✅ Завжди лінкуй на реальні сторінки послуг
4. ✅ Для кожного міста — лінкуй на свою міську сторінку
5. ✅ Додавай реальні Unsplash фото (тільки cleaning-relevant)
6. ✅ Використовуй ВЧ ключ у H1, НЧ ключі — у H2 і H3
